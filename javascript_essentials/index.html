Lesson 1: Variables, Data Types, and Operators

Variables

Variables are containers for storing data. Use let, const, or var to declare variables.

let name = "Alex";
const age = 30;
var isStudent = true;

Data Types

JavaScript supports various data types such as Strings, Numbers, Booleans, Null, Undefined, and Objects.

// String let text = "Hello World";
// Number let number = 42;
// Boolean let isTrue = false;
// Null let empty = null;
// Undefined let notDefined;
// Object let person = { name: "Emma", age: 28 };

Operators

Operators are used to perform operations on values like Arithmetic, Comparison, and Logical operations.

// Arithmetic operators let sum = 5 + 10;
let difference = 20 - 5;
let product = 4 * 3;
let quotient = 10 / 2;

// Comparison operators let isEqual = 5 === 5;
let isNotEqual = 10 !== 5;
let isGreater = 10 > 5;

// Logical operators let andResult = true && false;
let orResult = true || false;
let notResult = !true;

Lesson 2: Functions, Loops, and Conditional Statements

Functions

Functions group reusable code and use parameters to make functions dynamic.

// Function declaration function greet(name) { return "Hello, " + name + "!"; } // Function expression const multiply = function(a, b) { return a * b; }; // Try these functions greet("World"); multiply(5, 3);

Loops

Loops repeat code until a condition is met (e.g., For, While, and For...of loops).

// For loop let forResult = [];
for (let i = 0; i < 5; i++) { forResult.push(i); }

// While loop let whileResult = [];
let j = 0;
while (j < 5) { whileResult.push(j); j++; }

// For...of loop let array = ["apple", "banana", "cherry"]; let forOfResult = [];
for (let item of array) { forOfResult.push(item); }

Conditional Statements

Conditional Statements control flow based on conditions.

let score = 85;
let grade;

// If-else statement if (score >= 90) { grade = "A"; } else if (score >= 80) { grade = "B"; } else if (score >= 70) { grade = "C"; } else { grade = "F"; }

// Switch statement let day = 3; let dayName;
switch (day) { case 1: dayName = "Monday"; break; case 2: dayName = "Tuesday"; break; case 3: dayName = "Wednesday"; break; default: dayName = "Unknown"; }

Lesson 3: Arrays and Objects

Arrays

Arrays store multiple values in a single variable.

// Creating arrays let fruits = ["apple", "banana", "orange"];
// Accessing elements let firstFruit = fruits[0];
// Array methods fruits.push("grape"); // Add to the end fruits.pop(); // Remove from the end fruits.unshift("strawberry"); // Add to the beginning fruits.shift(); // Remove from the beginning
// Find the length let fruitCount = fruits.length;

Objects

Objects use key-value pairs to group related data.

// Creating an object let person = { name: "Sarah", age: 32, profession: "Developer", skills: ["JavaScript", "HTML", "CSS"] };
// Accessing properties let personName = person.name; let personAge = person["age"];
// Adding and modifying properties person.location = "New York"; person.age = 33;
// Nested objects let company = { name: "Tech Corp", employees: { developers: 50, designers: 30 } };

Lesson 4: ES6+ Features

Arrow Functions

A shorter syntax for writing functions.

// Traditional function function add(a, b) { return a + b; }
// Arrow function const addArrow = (a, b) => a + b;
// Arrow function with more statements const calculateArea = (width, height) => { const area = width * height; return area; };

Destructuring

Extract values from arrays or objects into distinct variables.

// Array destructuring const colors = ["red", "green", "blue"];
const [firstColor, secondColor] = colors;

// Object destructuring const user = { name: "John", email: "john@example.com", country: "Canada" };
const { name, email } = user;

Template Literals

String interpolation and multi-line strings.

// Old way const oldGreeting = "Hello, " + user.name + "! Welcome to " + company.name + ".";
// Template literals const newGreeting = `Hello, ${user.name}! Welcome to ${company.name}.`;
// Multi-line strings const multiLine = `This is a string that spans across multiple lines.`;

Adding Event Listeners

Event Delegation

Handle events for multiple elements with a single listener.

Click on any button below:

Click a color button...
// Event delegation - single event listener for all buttons const container = document.querySelector(".button-container"); const delegationOutput = document.getElementById("delegationOutput");
container.addEventListener("click", function(event) { // Check if the clicked element is a button if (event.target.classList.contains("delegate-btn")) { const color = event.target.dataset.color; delegationOutput.textContent = "Selected color: " + color; delegationOutput.style.color = color; } });
// Add a new button dynamically document.getElementById("addButton").addEventListener("click", function() { const colors = ["purple", "orange", "pink", "brown"]; const randomColor = colors[Math.floor(Math.random() * colors.length)]; const newButton = document.createElement("button"); newButton.classList.add("delegate-btn"); newButton.dataset.color = randomColor; newButton.textContent = randomColor.charAt(0).toUpperCase() + randomColor.slice(1); container.appendChild(newButton); });

Best Practices

Minimize DOM Access

Store references to frequently accessed elements in variables.

// Bad practice - repeatedly accessing the DOM function updateCount(count) { document.getElementById("counter").textContent = count; if (count > 10) { document.getElementById("counter").classList.add("highlight"); } }
// Good practice - store reference function updateCountBetter(count) { const counter = document.getElementById("counter"); counter.textContent = count; if (count > 10) { counter.classList.add("highlight"); } }
Why it matters: DOM operations are expensive. Minimizing them improves performance.

Use Event Delegation

Use a single event listener on a parent element to handle events for multiple children.

// Bad practice - individual listeners for each button document.querySelectorAll(".action-btn").forEach(button => { button.addEventListener("click", handleClick); });
// Good practice - event delegation document.querySelector(".button-container").addEventListener("click", function(event) { if (event.target.classList.contains("action-btn")) { handleClick(event); } });
Why it matters: More efficient for dynamic content, reduces memory usage, and simplifies code.

Avoid Inline JavaScript

Separate JavaScript from HTML for clean and maintainable code.

// Bad practice - inline JavaScript // <button onclick="doSomething()">Click Me</button>
// Good practice - external JavaScript // In HTML: <button id="actionButton">Click Me</button> // In JS file: document.getElementById("actionButton").addEventListener("click", doSomething);
Why it matters: Better separation of concerns, easier maintenance, and better caching.

Use classList for Class Manipulation

Use the classList API to add, remove, and toggle classes.

// Bad practice - manipulating className directly element.className = element.className + " active";
// Good practice - using classList element.classList.add("active");
element.classList.remove("inactive");
element.classList.toggle("highlight");
Why it matters: Cleaner code, prevents accidentally overwriting existing classes.
